### 1 Tạo action trong github
Bước 1: Mở github vào repository của bạn muốn deploy vô setting > Actions > Runners > Add runner 
- Khi đó bạn sẽ thấy các lệnh được hiển thị bạn để đó và bắt đầu vô vps
Bước 2: Vào VPS và tạo thư mục lưu trữ ứng dụng của bạn
```bash
cd ~
mkdir demitech.vuvandinh.id.vn
```
> Lưu ý: demitech.vuvandinh.id.vn là tên domain của bạn hoặc bạn có thể đặt bất cứ tên gì để dễ nhận biết

Bước 3: Chạy các lệnh ở bước 1 trong thư mục vừa tạo
```bash
# Bạn sẽ bỏ qua bước này và để mặc định
Enter the name of the runner group to add this runner to: 
# Nhập tên runner bạn muốn 
Enter the name of runner: [press Enter for interdata19082028] demitech.vuvandinh.id.vn
# Chọn định dạng deploy của bạn
This runner will have the following labels: 'self-hosted', 'Linux', 'X64'
# Thì hãy chọn self-hosted và nhấn enter
Enter name of work folder: [press Enter for _work]
# Nhập tên thư mục bạn muốn lưu trữ nên để mặc định
```
Bước 4: Cấu hình runner làm dịch vụ
```bash
./svc.sh install
``` 
Bước 5: Khởi động dịch vụ
```bash
./svc.sh start
```
> Lưu ý: Nếu lệnh trên bị lỗi hay kèm thêm sudo trước lệnh

Bước 6: Kiểm tra trạng thái của dịch vụ
```bash
./svc.sh status
```
Bước 7: Kiểm tra trạng thái của dịch vụ trong github
Vô github vào lại runner nếu thấy trạng thái là idle là thành công

Bước 8: Tạo file action trong github chọn new workflow vào thư mục .github/workflows tạo file nodejs.yml
```bash
# This workflow will do a clean installation of node dependencies, cache/restore them, build the source code and run tests across different versions of node
# For more information see: https://docs.github.com/en/actions/automating-builds-and-tests/building-and-testing-nodejs

name: Node.js CI

on:
  push:
    branches: [ "main" ] # Thay đổi branch của bạn muốn khi push lên sẽ thay đổi
  pull_request:
    branches: [ "main" ]
xo
jobs:
  build:

    runs-on: self-hosted # Thay đổi runner của bạn như bước 3 mình đã chọn self-hosted

    strategy:
      matrix:
        node-version: [18.x] # Thay đổi phiên bản nodejs của bạn
        # See supported Node.js release schedule at https://nodejs.org/en/about/releases/

    steps:
    - uses: actions/checkout@v4
    - name: Use Node.js ${{ matrix.node-version }}
      uses: actions/setup-node@v4
      with:
        node-version: ${{ matrix.node-version }}
        cache: 'npm'
    - run: npm ci
    - run: npm run build --if-present
```

<summary >(NOTE 03 09-10-2022) - Tạo Database trên VPS và cấp quyền truy cập cho máy tính từ xa</summary>
<div class="description">
#### 1. Cài đặt MySQL trên VPS
    1. Cập nhật hệ thống
    ```bash
    sudo apt update
    sudo apt upgrade
    ```
    2. Cài đặt MySQL nếu chưa có
    ```bash
    sudo apt install mysql-server
    ```
    3. Bảo mật MySQL
    ```bash
    sudo mysql_secure_installation
    ```
#### 2. Tạo database và cấp quyền truy cập cho máy tính từ xa
    1. Đăng nhập vào MySQL
    ```bash
    sudo mysql -u root -p
    ```
    2. Tạo database mới
    ```bash
    CREATE DATABASE ten_csd;
    ```
    3. Tạo user mới
    ```bash
    CREATE USER 'username'@'%' IDENTIFIED BY 'password';
    ```
    4. Cấp quyền cho user mới
    ```bash
    GRANT ALL PRIVILEGES ON ten_csd.* TO 'username'@'%';
    ```
    5. Lưu lại các thay đổi
    ```bash
    FLUSH PRIVILEGES;
    ```
    6. Thoát MySQL
    ```bash
    exit
    ```
#### 3. Cấu hình MySQL cho phép truy cập từ xa`
    1. Mở file cấu hình MySQL
    ```bash
    sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
    ```
    2. Tìm đến dòng bind-address và sửa thành
    ```bash 
    bind-address = 0.0.0.0
    ```
    3. Lưu lại file và khởi động lại MySQL
    ```bash
    sudo systemctl restart mysql
    ```
#### 4. Mở cổng Mysql trên tường nửa VPS
    1. Mở cổng 3306
    ```bash
    sudo ufw allow 3306
    ```
    2. Kiểm tra lại các cổng đã mở
    ```bash
    sudo ufw status
    ```
#### 5. Kiểm tra kết nối từ xa
    1. Dùng lệnh
    ```bash
    mysql -u username -h ip_address -p
    ```
    - username: là tên user vừa tạo
    - ip_address: là địa chỉ IP của VPS
    2. Dùng ứng dụng hỗ trợ kết nối MySQL như
    - MySQL Workbench
    - DBeaver
    - HeidiSQL
    - Antares SQL
    - Navicat

<summary >(NOTE 01 06-10-2022) - Các lệnh để thờng sử dụng trong github</summary>
<div class="description">
<p>1. Để tạo một repository mới trên github, bạn sử dụng lệnh sau:</p>
```bash
git init
```
<p>2. Để thêm tất cả các file vào staging area, bạn sử dụng lệnh sau:</p>
```bash
git add .
```
<p>3. Để commit các file vào repository, bạn sử dụng lệnh sau:</p>
```bash
git commit -m "First commit"
```
<p>4. Để đồng bộ các thay đổi lên github, bạn sử dụng lệnh sau:</p>
```bash
git push origin master
```
<p>5. Để tạo một nhánh mới, bạn sử dụng lệnh sau:</p>
```bash
git checkout -b feature-branch
```
<p>6. Để chuyển sang một nhánh khác, bạn sử dụng lệnh sau:</p>
```bash
git checkout master
```
<p>7. Để xem lịch sử commit, bạn sử dụng lệnh sau:</p>
```bash
git log
```
<p>8. Để xem trạng thái của các file, bạn sử dụng lệnh sau:</p>
```bash
git status
```
<p>9. Để xem sửa đổi của các file, bạn sử dụng lệnh sau:</p>
```bash
git diff
```
